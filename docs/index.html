Low‑level HTTP Server – Documentation

This library provides a lightweight, extensible HTTP server written in modern C++.

Server Creation & Listening

Instantiate a server with server::createServer(port) and start accepting connections via listen().

auto srv = server::createServer(8080);

srv.listen([]{ std::cout << "Listening…\\n"; });

Routing

Register handlers for all standard HTTP methods: GET, POST, PATCH, DELETE, HEAD, OPTIONS, PUT.

srv.get("/hello", [](Request& req, Response& res){
    res.send("Hello world");
});

Parameterized Routes

Routes may contain colon‑prefixed parameters (e.g. /users/:id). The framework extracts them into req.params.

srv.get("/users/:id", [](Request& req, Response& res){
    std::string id = req.params["id"];
    // ...
});

Middleware

Middleware runs before route matching. It receives a next handler to continue the chain.

srv.use("/", Method::ALL, [](Request& req, Response& res, NextHandler next){
    // Example: logging
    std::cout << req.method << " " << req.path << "\n";
    next(req, res);
});

CORS Support

Enable cross‑origin resource sharing with a single call.

srv.cors("/", "*", Method::ALL);

Static File Serving

Expose a directory under a URL prefix using staticDir.

srv.staticDir("/static", "./public");

Plugin System

Derive from ServerPlugin and register with registerPlugin(). Plugins can hook into the request lifecycle.

class Logger : public ServerPlugin {
public:
    void onBeforeRequest(Request& req, Response& res) override {
        std::cout << "[REQ] " << req.method << " " << req.path << "\n";
    }
};

srv.registerPlugin(std::make_unique<Logger>());

Request Parsing

The server parses HTTP headers and supports bodies of type JSON (via nlohmann::json), URL‑encoded forms, plain text, and binary data. Parsed data is available in req.body.

Response Helpers

Build responses fluently:

res.status(200)
   .setHeader("Content-Type", "application/json")
   .send("{\"ok\":true}");

Thread Pool

Internally the server uses a ThreadPool (defined in include/threadpool.hpp) to handle each client connection on a worker thread.

Content‑Type Mapping

Utility function contentTypeFromExtension() maps file extensions to proper Content‑Type headers for over 30 common MIME types.

IP Extraction & X‑Forwarded‑For

The server determines the client IP address using getpeername and respects the X‑Forwarded‑For header for proxied deployments.

Graceful Shutdown

Calling requestStop() or sending a SIGINT triggers a clean shutdown – the listening socket is closed, the thread pool is stopped, and plugins receive onShutdown callbacks.
